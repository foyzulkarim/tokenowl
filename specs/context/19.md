## Task

Add a GitHub Actions CI check that runs on every PR targeting `main` and fails if branch-only artifacts would be merged.

## Failure conditions

- Any file exists under `specs/` (any file type)
- Any root-level `.md` file is **not** on the allowlist

## Allowlist (root-level `.md` files permitted in `main`)

- `README.md`
- `CLAUDE.md`

Adding a new legitimate root-level `.md` (e.g. `CONTRIBUTING.md`) requires a deliberate allowlist update in the workflow file.

## Acceptance criteria

- CI check fails on a PR that adds a file under `specs/`
- CI check fails on a PR that adds a root-level `.md` not on the allowlist
- CI check passes when neither condition is triggered
- Workflow file lives under `.github/workflows/`

## Convention (not enforced by CI)

Before merging a PR, upload all `specs/` artifacts to the GitHub issue as comments or attachments, then delete the files. This preserves decision history without polluting the codebase.

## Notes

- Do NOT add `specs/` to `.gitignore` — Claude needs to read these files on feature branches.
- `docs/` is untouched — those are timeless project reference docs.
